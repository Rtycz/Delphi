Клиент-серверное приложение про такси
базу fdb открывать открывать IBEXPERTом
Пароль если он есть masterkey (более вероятно) или 1234 или 1
состоит из двух клиентов для разных устройств и одного сервера

начинается с открытия и запуска сервера
адрес запуска - 127.0.0.1
жмакаем запустить

далее открываем десктоп клиент
указываем путь к базе TAXI.fdb проще разместистить ее в C:\Delphi, чтобы не менять путь до нее на сервере и на клиенте
адрес сервера узнаем через cmd командой ipconfig и вводим его в строку с адресом
есть несколько типов операторов, отличающихся функционалом
посмотреть их можно вроде через TAXI.fdb
логин-пароль - admin-admin
открываем вкладку заказы и перетаскиваем заказы, они отображаются по кнопке обновления в андроид клиенте

открываем проект андроид, либо билдим заново, либо достаем apk и устанавливаем на телефон
логин-пароль юзера 2-2 - (Мария Копейкина)
адрес вводится в виде http://192.168.0.105:7777 где вместо "192.168.0.105" - адрес сервера
приложение вылетает если зайти в подробности заказа выйти и зайти заново, но все фиксится кнопкой обновить
если логин-пароль совпали то все заказы этого курьера отправятся в json файл и отображаются в main через 
OrdersListViewAdapter структурами OrderDataModel
Push уведомления через firebase не работают
FileHolder записывает и вытаскивает в json файл данные для автозаполнения при регистрации
HttpRequester ничего не делает и все что было в нем есть в main

десктоп клиент общается с сервером udp запросами,
а сервер с базой вроде как tcp-ip но это неточно, сервер Машин
при изменении информации одним оператором, изменения обновятся на экране того же оператора на другом устройстве с помощью протоколов udp
для этого на сервере есть клиент, а на клиенте есть сервер которые общаются через порт 11000
и при изменении заказов на клиенте отправляются данные на сервер, которые заставляют вызвать обновление заказов на другом клиенте тоже
сервер и мобильное приложение общаются http.post запросами
